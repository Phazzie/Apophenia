# Gemini Coding Assistant Instructions for the Apophenia Project

This document provides essential context and guidelines for the Gemini AI coding assistant to ensure all contributions are high-quality, consistent, and aligned with the project's architecture.

## 1. Project Overview: Apophenia

Apophenia is a next-generation psychological horror game that uses a sophisticated, multi-layered AI system to create a deeply personal and dynamic narrative. The game's core is built around the idea of a malevolent, meta-aware AI that actively manipulates the game world, the UI, and even the player's perception of reality.

- **Primary Goal:** To create a sense of cosmic dread and psychological unease.
- **Core Mechanic:** The game progresses through a series of `GameCommand` objects generated by AI, which are then executed by the client.
- **AI System:** A hybrid model using Gemini and Grok for narrative generation, with a suite of specialized "AI Engines" that modulate the experience.

## 2. Core Technologies

- **Language:** TypeScript
- **Framework:** React with Vite
- **State Management:** Zustand
- **AI Services:**
  - `@google/generative-ai` (for Gemini models)
  - A custom `xaiClient` (for Grok models)
  - `unifiedAIService.ts` acts as a router between these models.
- **UI:** React components styled with standard CSS.

## 3. Architecture & Coding Principles

### Principle 1: Command-Based Game Loop

The entire game is driven by an array of `GameCommand` objects. The AI **must** always return a valid JSON array of these commands.

**`GameCommand` Discriminated Union:**
```typescript
type GameCommand =
  | { type: 'displayText', payload: { content: string; segmentId: string; } }
  | { type: 'generateImage', payload: { prompt: string; segmentId: string; } }
  | { type: 'displayChoices', payload: { choices: Choice[]; intrusiveThought?: Choice; } }
  | { type: 'updateWorldState', payload: Partial<WorldState> };
```

- **Rule:** When generating code that produces commands, always use this structure.
- **Rule:** Every `displayText` and `generateImage` command **must** have a `segmentId`. This is critical for state synchronization and image caching.

### Principle 2: Zustand for State Management

The global state is managed in `src/stores/worldStateStore.ts`.

- **To Read State:** Use `useWorldStateStore.getState().worldState`.
- **To Update State:** Do **not** update the state directly. Instead, generate an `updateWorldState` command. The game loop will process this command and update the store.

### Principle 3: TypeScript Excellence

- **Avoid `any`:** Use specific types wherever possible. The codebase is strongly typed.
- **Use Nullish Coalescing (`??`) and Optional Chaining (`?.`):** Write modern, clean TypeScript.
- **Leverage Utility Types:** Use `Partial`, `Pick`, `Omit`, etc., to create precise types.

## 4. How-To Guides

### How to Add a New AI Engine

1.  **Create the Engine File:** Create a new file in `src/services/ai/engines/`. Name it descriptively (e.g., `NewHorrorEngine.ts`).
2.  **Define the Class:** The class should encapsulate a specific horror mechanic. It can read from `worldState` but should not modify it directly.
3.  **Export an Instance:** In `src/services/ai/engines/index.ts`, create and export a singleton instance of your new engine.
4.  **Integrate into the Game Loop:** In the main game logic file, import the new engine instance and call its methods at the appropriate time to influence the AI prompts or modify game behavior.

### How to Introduce a New Game Mechanic

1.  **Define the Command:** If the mechanic requires a new UI interaction, add a new type to the `GameCommand` discriminated union in `src/types.ts`.
2.  **Update the Processor:** In the client-side game loop where commands are processed, add a `case` for your new command type and implement the logic to handle it.
3.  **Instruct the AI:** Update the core AI prompts (`grok_next_step_prompt.txt` or the prompts in `genkit.ts`) to teach the AI how and when to use your new command. Provide clear instructions and examples.
